MapBuilder Open Source GML Viewer for OGC's OWS4 testbed.

About

The Open Source GML Viewer renders GML2, GML3 and WMS layers as described by an OWS Context document within a Web Browser.

The Viewer is intended to be distributed with recent GML datasets to allow uses to easily view the datasets they have been provided.

Examples

  • MSD3 dataset (The MSD3 dataset is not stored in the Mapbuilder repository due to license restrictions)
  • Tasmania dataset

Features

  • Can be executed by opening a web page from your local drive. It runs inside a recent web browser without plugging.
  • Browsers supported include Internet Explorer 6 and Firefox 1.5+
  • Renders GML2 and GML3 layers are SVG or VML.
  • Can render hundreds of GML features easily, and thousands of features sluggishly. (Firefox is much faster than IE).
  • Renders WMS layers.
  • Layers are described using a OWS Context document.
  • Feature colors are specified using SLD in with OWS Context.
  • Free to use and modify under an Open Source license.

Build your own client

This is a very quick guide for building your own GML Viewer. It makes the bold assumption that you are already familiar with HTML, OWS Context, and GML. Let's assume you want to build an African dataset.

  1. Copy tasmania.html to africa.html. Edit africa.html and find
    // URL of Mapbuilder configuration file. var mbConfigUrl='tasmaniaConfig.xml';
    Replace 'tasmaniaConfig.xml' with 'africaConfig.xml' (which you are about to create).
  2. Copy tasmaniaConfig.xml to africaConfig.xml. Edit africaConfig.html and find
    <OwsContext id="mainMap"> <defaultModelUrl>tasmania/TasmaniaContext.xml</defaultModelUrl>
    Replace 'tasmania/TasmaniaContext.xml' with 'africa/AfricaContext.xml' (which you are about to create).
  3. Insert all your GML files into your directory somewhere, e.g.: africa/gml/* . Ensure all GML files have a .xml extension, not a .gml extension! When web browsers load local files, they don't recognise they only recognise XML files that have a .xml extension. Without it your layers will not load.
  4. Create an AfricaContext.xml document, use TasmaniaContext.xml as reference. It should reference your GML files.

If you have problems, try emailing the Community Mapbuilder uses email list. We are a helpful lot.
