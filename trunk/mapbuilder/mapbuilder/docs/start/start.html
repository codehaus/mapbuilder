Getting started

There are several common steps for coding a mapbuilder application. In the following code examples, paths starting with "/" are relative to your server root.

  1. Create the HTML for the web page: All that is required is to assign an ID attribute to the HTML elements in the page where mapbuilder widgets will be inserted. For example: 
    <div id="mbId"></div>
    where "mbId" corresponds to a widget ID in your configuration file.
    Note: don't use the empty node form for mapbuilder elements (i.e. <div id="mbId"/>). This is because some browsers do not recognize the empty tag as being closed.
  2. Create your mapbuilder configuration file: Copy an existing configuration file or create a new one. Adjust the URL of models to point to the documents you wish to load. Then include the widgets desired, setting it's ID attribute to the corresponding HTML element in your page. Finally, add in any tools desired.
  3. Include the mapbuilder Javascript library: the web page's <HEAD> element must include a reference to the mapbuilder library JavaScript code. The library can be installed on the same server as the page using it: 
      <script type="text/javascript" 
  src="/mapbuilder/lib/Mapbuilder.js"></script>
    OR, it can be referenced from another server (not tested yet, might be some issues to be resolved): 
      <script type="text/javascript" 
  src="http://mapbuilder.sourceforge.net/mapbuilder/lib/Mapbuilder.js"></script>
    To install the library on your server, (full details to be worked out) unzip the mapbuilder library into an HTTP directory on your server.
  4. Initialize the mapbuilder library: This involves adding a <SCRIPT> block to your page <HEAD> to initialize the mapbuilder library, passing the URL of the configuration file created in step 2. 
      <script>
  Mapbuilder("/myApp/myAppConfig.xml");
  </script>
  5. Initialize the application: the page's BODY onload event handler must call mapbuilder's body onload event handler. This can be chained with other application onload handlers if required: 
      <body onload='mbConfig.init(); myApp.onLoadHandler()'>
    ..rest of page body...
  </body>
