Mapbuilder Demo Pages

Simple Map Viewer
A minimal map implementation showing a map embedded in a page.
Basic Map Viewer
A basic map viewer implementation. This is a good demo to copy when building a new application.
Demis Maps Viewer
A Demis maps viewer which allows a user to switch maps Google style.
WMS Time Series
Shows maps as a time series. Zoom in to a region of interest.
Complete configuration
Not pretty to look at but shows integration of most components. Intended for testing extensions to the code base.
Feature Entry
Shows editing and updating of GML features and uploading to a Transactional Web Feature Service (WFS-T). This demo accesses data from a Geoserver installation installed in the same Java Servlet Webapps directory as Mapbuilder.
Context Editor
Creation and editing of Web Map Context documents from searching and browsing a registry.
Live RSS/Atom Feeds with georss extension
A small application initially demonstrated at OSG'05 that showed how to extend Mapbuilder.
This one processes a live geoRSS feed. While this is Atom feed variation of it (using georss pro extension) from georss.org
Live Flickr RSS/Atom Feed of The MapBuilder Community
This application enhances the previous demonstration of rss feeds. It adds images on the map, popups, SLD processing... Rss layers are added dynamically to the legend and can be tuned on/off
Ship Tracks RSS/Atom Feeds and Vector Graphics rendering on the client side
This application enhances the previous demonstration of rss feeds even more. It adds tracks, polygons on the map, popups, SLD processing to demonstrate vector graphics on the client side. It uses VML on IE and SVG on Firefox 1.5. For other platforms, it will need more work. This is still experimental and a little incomplete but the idea is there.
  • Browsers supported are Internet Explorer 6 or later, Mozilla 1.3 or later, Netscape 6 or later.
  • Some functionality requires the server proxy and serialize scripts in the configuration file. Implementations of these are available for PHP enabled servers and Java Servlet Container (like Tomcat).
