Welcome

Maybe you're looking for a free, standards-based mapping solution that's easy both to set up and to tweak to your needs.  Or else you've heard about the Web Feature Server standard for geographic data delivery over the internet and want to see it in action.  Data streams are fine and well, but what about turning those numbers into maps?


Example of GeoClient user interface

In either case, GeoClient is written with you in mind.

That said, this initial release is just a preview--enough for demonstration purposes, but not yet ready for full-scale use.  In particular, this version only works with its own simple testing web feature server and not - yet - with GeoServer.

Part of the GeoServer project, GeoClient is a graphical software for online mapping.  GeoClient is designed to work as a "client" to a server operating on the Open GIS Consortium's Web Feature Server standard.

The GeoServer project's mission is to enable greater geographic interoperability by implementing OpenGIS standards and lowering the barriers to entry for geographic data providers and users.

Written in Scaleable Vector Graphics (SVG) and ECMAscript/JavaScript, GeoClient is designed to run "natively" in a web browser, once SVG support is available.  In the meantime, it uses the Adobe SVG Viewer plugin.  GeoClient is a parallel development to Java-based open source WFS client initiatives like those within the GeoTools project.

As well as the SVG-based client, GeoClient has two accompanying components:

  • ToWKT, an ArcView 3.x extension for preparing and converting geographic data and settings, and
  • GeoServerLite, a simple Web Feature Server written in the PHP scripting language.

The ToWKT extension exports data into a MySQL database in the "well known text" format defined in the OGIS Simple Features for SQL specification, while the PHP-based WFS connects to the MySQL database. Easy and quick to set up, these tools are designed for testing purposes and for basic use on operating systems other than Linux. (GeoServer currently requires a PostGIS installation, which runs on Linux or the Cygwin Unix emulation for Windows.)  Also, GeoServerLite requires no special software on the web server beyond the free, open source, and broadly available MySQL database and PHP scripting. This means that the approach can be used by organizations lacking a dedicated server.  

While GeoClient is limited for now to use with GeoServerLite, for production-level applications it should be used with GeoServer (or with another similarly full-featured and robust WFS).  GeoServer support is a primary aim of current development.

To get started, download the file geoclient-0.10.tar.gz from the GeoServer Sourceforge downloads page.

Features
Currently available:
  • Cross-browser support (through Adobe SVG Viewer plug-in).
  • Issues requests as per WFS specification.
  • Parses (interprets and transforms) incoming geography markup language (GML) data into graphic representation in SVG.
  • Displays x and y coordinates on mouse move.
  • Features can be labelled automatically.
  • Implements custom pan and zoom controls.
  • Layer display can be switched off and on.
  • ID function: clicking a feature that has attributes brings up an attribute table loaded from local data.
  • Features (e.g., text labels, circle radius, line width) scale on zoom [incomplete support].
  • Scale-dependent data loading: Map features fitting into the current map extent and set to display at the current scale are loaded dynamically along with their attributes on zoom and and pan. [GeoClient support in place, but filtering not yet supported by GeoServerLite.]

Under development:

  • Geographic data input and editing: interface for adding new point, line, or polygon features.
  • Map data can be dynamically selected or filtered through a query interface.
  • On zooming, layers and their legend symbols are displayed or hidden based on minimum and maximum scale settings for the layer.
  • Optional linking of features to external files or online databases (e.g., click on a contaminant release site to bring up that site's record in a federal online database).
Limitations
  • Uses proprietary Adobe functions printNode(), postURL(), and parseXML().
  • Not fully tested on all operating systems and browser versions.
Requirements
  • Computer running ArcView GIS 3.x (for the export script).
  • Geographic data in any format supported by ArcView.
  • A web server with MySQL and PHP (XML enabled).
  • A web browser with the Adobe SVG Viewer plug-in.
Quick Start
If you want to get started right away, here are some very basic instructions.
  1. Ensure you have the following software requirements in place: ArcView 3.x (if prior to 3.1, download and install "dialog designer" extension from ESRI website); MySQL; web server (e.g., Apache); PHP (note: must have XML support); and Adobe SVG Viewer browser plugin.
  2. In ArcView 3.x, load the extension toWKT.  Create a new view, click the blue diamond tool to export.
  3. In MySQL, create a new database and import the dump file created by the toWKT extension:

    >mysql
    mysql> create database databasename;
    mysql> exit
    >mysql databasename < [path to dumpfile]dumpfile.sql

    where databasename is the name of a new database, [path to dumpfile] is the path to the dumpfile created by toWKT, and dumpfile.sql is the dumpfile.
  4. Copy all files in the distribution to a php-enabled web directory, edit the file connect.php to reference your new MySQL database, and open the page http://yourserver/pathtogeoclient/geoclient.svg in a web browser (where "yourserver" is the address of your server and "pathtogeoclient" is the path to the GeoClient files on the server).

For detailed instructions:

  • Prepare and export data following ToWKT user instructions.
  •  Set up the simple web feature server following GeoServerLite user instructions.
  • Complete GeoClient setup as follows.
Setup
Once you have created a dataset using the ToWKT extension and set up your GeoServerLite server, complete your GeoClient setup:
  • Move the files geoserver.svg. geoserver.js, and legend.js to the same web server directory where you put the GeoServerLite files.
  • Test the installation by opening the page
    http://yourserver/path_to_geoclient/geoclient.svg
    You should be prompted for the path to the web feature server.  If you have installed GeoClientLite in the same directory as GeoClient, just accept the default value.  You'll also be asked if you wish to display requests and responses.  If you choose OK, requests by the client and responses from the server will be displayed as alerts and prompts; if not, the exchanges will be silent.
  • Use the tools on the toolbar to navigate around the application,

Known Issues
  • In some versions of Microsoft Internet Explorer, browser freezes if a request to another URL is sent after application loads.  A similar error prevents ID information from appearing (error message is given instead).
