Welcome

GeoServerLite is a simple Web Feature Server (WFS) written in the broadly available PHP scripting language and based on a MySQL backend database.  For production-level applications, we recommend using instead GeoServer, our robust open source WFS written in Java.  However, GeoServerLite may be useful for the following purposes:

  • For testing or demonstration, or for use by organizations lacking a dedicated server.  GeoServerLite uses the broadly available MySQL database and PHP scripting language, and so can be implemented quickly in many web hosting environments where the installations required for GeoServer would not be feasible.
  • For use in conjunction with GeoServer or another WFS to provide the "vendor-specific" functionality required by GeoClient.  GeoClient, an online mapping component of the GeoServer project, requires several "vendor-specific" server capabilities.  Although these capabilities are not - yet - available in GeoServer, it is easy enough to use GeoServer (or another WFS) and GeoServerLite in combination, thus providing both the robust data serving of GeoServer and the extended functionality of GeoServerLite.  This is done through coding in the Capabilities document served by the WFS (see detailed instructions in the GeoServerLite Developer documentation).

GeoServerLite is bundled for distribution with GeoClient and toWKT (a data preparation tool).  To get started, download the file geoclient-0.10.tar.gz from the GeoServer Sourceforge downloads page.

Features
  • Basic support for WFS capabilities.
  • Several vendor-specific capabilities to enhance mapping capabilities when used with GeoClient.
Limitations
  • Not fully compliant with WFS specification.
  • Only very basic geographic filtering available. 
Requirements
  • A web server with MySQL and PHP (XML enabled).  See XML Parser Functions in the PHP manual at http://www.php.net for setup instructions..  On Linux/Unix servers, PHP must be compiled with the --with-xml argument.  Note: it may be necessary to add an additional header type to the server settings, so that the server provides information necessary to client web browsers on SVG files.  
  • Geographic data in a MySQL database.  See toWKT for instructions on generating a database from ArcView. 
Setup
  1. Generate a dataset using the ToWKT extension, following the ToWKT user instructions.
  2. In MySQL, create a new database and import the dump file created by toWKT.
    • Create a new database with a name in the format
      view_name
      (that is, the name of the view from which data was exported, with underscore _ characters substituted for spaces). Example:
      mysql> create database mexico;
      mysql> exit
    • Using the mysql command, import the dump file generated in step 1 into the MySQL database just created. Example on a Windows machine:
      c:\mysql\bin> mysql mexico < mexico.sql
      (this example assumes that the file mexico.sql has been copied from the directory it was originally written to, e.g., c:\temp, to the MySQL bin directory, which is commonly c:\mysql\bin on a Windows machine.)
    • Ensure that a new or existing MySQL user account has read and write access to the new database.
  3. Copy the files *.php from the geoclient distribution to a PHP-enabled directory on a web server.  (Ensure that PHP has been compiled with XML support).  
  4. Using a text editor, open the file connect.php. It will look something like this:
    <?
    $db = mysql_connect("localhost", "root","password");
    mysql_select_db("view_name",$db);
    ?>
    Ensure that the values in the line starting "$db" reflect the correct MySQL server address and the user name and password for a user with write access to the database and that the database name in the line beginning "mysql_select_db" is the name of the database you just created. Note that all values are case-sensitive (so pay attention to capitals/lower case). Make any necessary changes and save and close connect.php.
  5. Finally, complete your GeoClient setup according to the GeoClient user instructions.
