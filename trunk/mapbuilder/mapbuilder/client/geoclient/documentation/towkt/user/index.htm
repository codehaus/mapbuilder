Welcome

Part of the GeoServer project, ToWKT is an extension to the GIS software ArcView that prepares data for online mapping using the GeoClient software and its accompanying server software GeoServerLite.

GeoClient is a graphical "client" for working with geographic data served by GeoServer, GeoServerLite, or another server operating on the Open GIS Consortium's Web Feature Server standard.

The GeoServer project's mission is to enable greater geographic interoperability by implementing OpenGIS standards and lowering the barriers to entry for geographic data providers and users.

ToWKT exports data into a MySQL database in the "well known text" format defined in the OpenGIS Simple Features for SQL specification.  ToWKT also optionally exports data into two other formats: Scaleable Vector Graphics (SVG) and Geography Markup Language (GML).

ToWKT is bundled for distribution with GeoClient and GeoServerLite .  To get started, download the file geoclient-0.10.tar.gz from the GeoServer Sourceforge downloads page.

Requirements
  • Computer running ArcView GIS 3.x.  If you have a version of ArcView 3 prior to 3.1, you will need to download and install the Dialog Designer extension before using ToWKT.  This is a free download from the ESRI site.
  • Geographic data in any vector format supported by ArcView.
  • A web browser with the Adobe SVG Viewer plug-in.
Using the Extension
  • Copy the file toWKT.avx from the GeoClient distribution to your ArcView 3.x extensions directory (typically the ext32 directory in your ArcView install path).
  • In ArcView, create a new project (or open an existing one) and load the extension (File > Extensions > toWKT interface extension).  This will add a blue diamond button to all View objects.
  • Prepare a view for export:
    • Create a view, or open an existing one.
    • Make sure you've set the view's map units (under the View > Properties menu).
    • Set Maximum Scale and/or Minimum Scale for any themes for which you want scale-dependent display--you'll find these settings under the Theme > Properties menu.  (As these settings are not yet supported in the GeoServerLite software, any layers with defined scales that don't display at the initial load scale won't be available.)
    • Make active the theme or themes you wish to export.
  • The following window appears.



    This window allows you to adjust settings for each layer before exporting by clicking on the values.  For example, click on one of the NameField values (e.g., in the example above, "Type") to change it. Several of the settings are not yet used in the GeoServerLite/GeoClient software.  Assuming that you've already got the display of a layer the way you want it, in most cases just making sure the NameField values are correct will be enough preparation before exporting.
  • Click the Export button.  You'll be prompted for a directory to save to--and that's it.
  • Optionally, you can also export data directly to either Scaleable Vector Graphics or Geography Markup Language.
    • For SVG export, click the "SVG test" button.  You'lll be prompted for the location of the source.svg file, part of the GeoClient distribution.  To publish, copy the resulting file to an internet directory along with the files toSVG.js.gz, legend.js.gz, id.html, help.html, and geoclient.css from the GeoClient distribution.
    • For GML export, click the "GML test" button.  GeoClient does not yet support direct viewing of the resulting file.
